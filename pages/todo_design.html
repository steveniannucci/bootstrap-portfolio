Description

This is the design I came up with for the to-do app before I started developing it. I decided to include the login (first slide) and registration (second slide) screens in the slideshow since I needed to implement these into the design. The task list menu by default has no tasks on it and the screen would prompt the user to add one (third slide). On this same screen, there is a three dot menu where the user could either add a task or sign out of the app. If the user chooses to add a task, they would be met with the New Task screen (fourth slide). To create the task, the user needs to input the task name and deadline. When developing the app, I was able to keep this format for adding tasks but I had to include it on the task list screen instead. On the edit task screen (fifth slide), the user can edit the task name and deadline. They could also delete a task or mark it as completed. If a task is marked as completed, it would be moved to the completed tasks screen (sixth slide). On this screen, the checkbox within the task container would be checked to indicate that the task is completed. If the box is unchecked, it would be moved back to the task list screen and vice-versa. The user can also toggle between these screens via the dropdown menu. When the user is done with the app, they can sign out and they would be met with the sign out screen (seventh slide) where they can tap on the option to login and be brought back to the login screen.
